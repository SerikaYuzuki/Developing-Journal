# 大学講義ノートの編集方針

`posts/univ_lecs/` 配下の講義ノートを新規作成・再構成・過去問統合するときの共通ルール。読者が**最初から順に読めば初見の問題にも取り組め、最後に模範解答で答案を確認できる**ことを目標にする。

## 1. 一次資料を起点にする

- PDF、配布資料、板書写真などの**元資料を優先**する。OCRテキストは検索・下書きには使えるが、数式、添字、符号、空欄、単位、表の値は必ず元資料で照合する。
- OCRが壊れた式を推測で確定しない。判読できないときは元PDFを高解像度で確認し、それでも不明なら不明点を残して確認を求める。
- 年度・元の問番号・「統合元」「出題対応」などは、学習用本文では原則として書かない。必要なのは出典の並びではなく、概念のつながりである。
- 同じ概念・導出・計算を問う問題は一つへ統合する。片方にしかない条件、数値例、よい問い方は統合問題へ残す。

## 2. 読む順番は「問題 → 理論 → 解答」

各演習は次の骨格にする。

```markdown
#### 問題X1 短く内容が分かる題名

問題文。必要なら資料図・表をここに置く。

::: {.callout-note collapse="true"}
##### 理論X1：なぜこの式・現象になるか

前提、図、導出、直感、典型的な誤り。
:::

::: {.callout-tip collapse="true"}
##### 解答X1

試験答案として必要な仮定、式、結論だけを簡潔に書く。
:::
```

- 問題文は、自力で考えるための入力だけにする。長い説明・導出・解法のヒントは理論欄へ移す。
- 理論欄では、式の前提、何を保存・近似しているか、境界条件、符号、単位を説明する。初学者が式を「暗号」と感じないよう、**なぜ**・**どのように**を先に言葉で示す。
- 直感的な比喩は有効だが、物理量の対応と比喩が破綻する点も明示する。たとえば電荷中性条件は「キャリヤの出どころ」ではなく、最終状態の正負電荷の帳簿として説明する。
- 解答欄は理論の要約ではない。問題の問いに直接答え、採点される仮定・式・数値・単位・結論を過不足なく置く。

## 3. 概念の順に並べる

- 元資料の年度順・配布順をそのまま採用しない。後の問題を解くのに必要な概念を先に置く。
- 典型的には「観察・定義 → 基本法則 → 導出 → 計算 → 応用・設計」の順にする。
- 複数章をまたぐ連鎖問題は、末尾などに総合問題として置く。前の各節で道具を学び、総合問題で式同士がつながることを確認できるようにする。
- 定数表・公式集は本文の後ろに付録として置く。公式には適用条件・符号・よく混同する量を添える。

## 4. 図・表・コードの置き場所

- 設問が図を読み取らせるなら、元資料の図を**問題文の直後**に残す。キャプションと `fig-alt` を付ける。
- 説明のための模式図、導出を助けるグラフ、Pythonで再現した図は**理論欄の中**へ置く。問題の答えを先に見せない。
- 図は「何を比較し、何を読ませるか」が一目で分かるようにする。軸、単位、凡例、代表値、色以外の区別を必ず確認する。
- Python図には `label`、`fig-cap`、`fig-alt` を付ける。数値例は問題の条件と同じ値を使い、模式図なら模式図であることを明記する。

## 5. 数式・記号・単位

- `physics` パッケージを前提とし、積分の微小量は `\dd`、常微分は `\dv`、偏微分は `\pdv` を使う。`\frac{\dd y}{\dd x}` や `\frac{\partial y}{\partial x}` の旧式表記を混在させない。
- 記号を最初に定義し、記事内で意味を変えない。特に密度、濃度、電場、電位、エネルギー、電荷の符号を統一する。
- 数値解答には単位を付ける。計算で使った近似（完全電離、非縮退、低注入、一次元、定常状態など）は答案にも一行で示す。
- 与えられた定数表が丸めによって完全には整合しない場合、どの値を優先して計算したかを明記し、図から過剰な有効数字を読まない。

## 6. 編集時の確認手順

1. 元資料から実際の設問、資料図、表、空欄、定数を抽出する。
2. 重複をまとめ、概念順の見出しと問題の対応を設計する。
3. 各問題を「問題 → 折りたたみ理論 → 折りたたみ解答」にする。
4. 数式表記、記号、単位、図の参照先を全体検索でそろえる。
5. 対象記事をレンダーする。

```bash
uv run quarto render 'posts/univ_lecs/<学年・科目>/<記事>.qmd'
git diff --check
```

6. 生成HTMLで、見出し順、問題・理論・解答の対応、内部リンク、数式、画像、折りたたみ、デスクトップと狭い画面での横はみ出しを確認する。
7. `_site/` の生成物もソースと一緒にコミットする。このリポジトリでは `_site/` が公開成果物である。

## 7. 完成条件

- 初見の読者が問題文を読んだ後、理論を開けば解答へ到達できる。
- 理論を閉じた状態でも、問題が自力演習として成立する。
- 解答だけを開けば、試験答案に必要な内容を短時間で復習できる。
- 元資料の誤読・OCR由来の崩れ・重複問題・出典管理用メタ情報が、学習の流れを妨げない。
